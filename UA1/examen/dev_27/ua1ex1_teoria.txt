PARTE TEÓRICA

a) Definición de proceso. 1 punto.  

Un proceso es un programa en ejecución


b) Utilizando tus propias palabras, indica la relación entre hilos y procesos. 3 puntos.

Puede haber varios hilos en ejecucion dentro de un mismo proceso, es decir los hilos contienen toda 
la información del proceso y trabajan para resolver un mismo problema.


c) Utilizando tus propias palabras, indica las características de la programación concurrente y sus ámbitos de aplicación. 3 puntos.

La programación concurrente es la que permite la ejecucion de varios procesos al mismo tiempo, este tipo
de programación solo permite la ejecución de un hilo por proceso.

Tambien se puede considerar programación concurrente cuando las instrucciones se intercalan.

Este tipo de programacion esta pensada para sistemas monoprocesador y necesita un S.O que controle la comunicacion y sincronizacion

Otra característicasde la programación concurrente es que las instrucciones se intercalan


d) Utilizando tus propias palabras, indica las diferencias entre programación paralela y programación distribuida, sus ventajas e inconvenientes. 3 puntos.

La programación paralela es la que esta pensada para ejecutarse en un sistema multiprocesador

Sus ventajas son: 
-Ejecución simultánea
-Puede resolver problemas complejos
-Menor tiempo de ejecucion

Sus desventajas son:

-Programas dificiles de escribir
-Es más dificil integrar un acceso a datos
-la comunicacion y sincronizacion es mas compleja.

La programacion distribuida es aquella en que sus software y hardware estan distribuidos, es decir, que sus componentes se interconectan
mediante una red.

Sus ventajas son:

-Facilidad de compartir datos y recursos
-Mayor flexibilidad a cambios
-Tiene alta disponibilidad

Sus inconvenientes son:
-Es mas complejo de utilizar y gestionar
-Al usar redes como medio de comunicacion es propenso a tener perdida de paquetes y ataques DDOS.
